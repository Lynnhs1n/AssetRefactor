这份说明书为 AssetRefactor Pro (GUI 版) 的开发提供了清晰的蓝图。该软件旨在通过精准的元数据提取和智能聚类算法，解决摄影师与影视从业者在 NAS 或本地存储上的资产归档痛点。



 AssetRefactor Pro 软件需求规格说明书 (SRS)

版本： 1.0 (2026 Build)
定位： 专业级数字资产重构与命名工具
技术栈： Python 3.x, PyQt6, Exiftool, Hashlib



 1. 界面与品牌识别 (Branding & UI)
 1.1 视觉规范
 软件名称： AssetRefactor
  核心组件：  源路径/目标路径选择器。
     `ProjectCode`与 `SubTopic` 输入框。
     实时控制台：滚动展示当前处理的文件名、校验状态及错误日志。
     进度条： 总任务进度的可视化反馈。

 1.2 增强型导入预览
 多维信息展示： 选择源路径后，弹出/嵌入预览窗口。
 媒体支持：  图片： 显示缩略图及 EXIF 拍摄时间。
     视频： 支持关键帧预览。
 交互筛选： 用户可在导入前勾选/去选特定文件，确保“同项目”资产的一致性。


 2. 核心命名逻辑与目录结构
软件采用日期驱动的命名体系，根目录格式为：`{YYYYMMDD}_{ProjectCode}_{SubTopic}`。

 2.1 资产分类规格
导入时，软件需自动创建并分发文件至以下结构：
 `/imgRaws/`: 存放图片资产 (`.ARW`, `.CR3`, `.DNG` 等) 及关联的 `.XMP` 侧边文件。
 `/vidRaws/`: 存放视频资产 (`.MP4`, `.MOV`)。
 `/Masters/`: 预设二级空目录。
 `/Exports/`: 预设二级及三级空目录。

 2.2 重命名逻辑公式
所有文件名均需通过正则截取原文件末尾 4 位数字作为序列号（`Original4Digits`）。

| 场景 | 目标路径 | 文件名公式 |
| : | : | : |
| A. 序列文件夹导入 | `/imgRaws/[原文件夹名]/` | `{YYYYMMDD}_{ProjectCode}_{SubTopic}_{原文件夹名}_{Original4Digits}.{ext}` |
| B. 散碎文件导入 | `/imgRaws/` | `{YYYYMMDD}_{ProjectCode}_{SubTopic}_{Original4Digits}.{ext}` |

 2.3 智能聚类 (Smart Clustering)
针对散碎文件，软件需执行实时扫描：
 触发条件： 若文件间“时间间隔 < 3秒”且“文件名逻辑连续”。
 处理动作： 自动创建名为 `AutoSeq_N` 的子文件夹进行归类，并按公式 A 进行重命名。



 3. 关键技术要求

 3.1 元数据处理 (Exiftool)
 核心调用： 必须使用 `exiftool` 提取 `DateTimeOriginal` 作为 `{YYYYMMDD}` 的唯一来源。
 容错机制： 启动时检查环境。若未发现 `exiftool`，弹出文件选择对话框引导用户指定其可执行文件路径。

 3.2 文件正则处理
 精准截取： 使用正则表达式 `(\d{4})\.[a-zA-Z0-9]+$` 锁定原文件名末尾的 4 位数字，避免因前缀混淆导致的编号错误。

 3.3 数据安全 (Data Integrity)
 哈希校验： 针对所有跨设备（如移动硬盘至 NAS）的传输操作，必须在移动/复制完成后进行 Hash (MD5/SHA1) 比对。
 校验失败处理： 若校验不一致，停止后续操作并在日志中以高亮红色标记该资产。



 4. 交互逻辑流程
1.  初始化： 设置 `ProjectCode` 与 `SubTopic`。
2.  源扫描： 用户通过预览界面确认待导入资产。
3.  规则匹配： 软件自动判断是“序列导入”还是“散碎聚类”。
4.  执行任务：
     创建目录结构。
     读取 EXIF 确定日期。
     重命名并移动文件。
     进行哈希校验。
5.  结束： 状态栏提示“任务完成”，生成简要报告。
